# Sprint docs (временные)

Эта папка — место для **временных документов спринта**: черновики, заметки,
decision‑log’и по конкретной задаче, быстрые расследования, списки проверок.
Цель — не захламлять “каноничные” документы в `docs/`, но сохранять контекст,
пока работа активна.

## Правила

- **Срок жизни**: документы здесь считаются временными. После завершения задачи:
  - либо **перенеси полезное** в каноничный документ (`docs/SECURITY.md`,
    `docs/ARCHITECTURE.md`, `docs/DEPLOY.md`, `docs/UX_FLOW.md`, …),
  - либо **удали файл**, если он больше не нужен.
- **Не дублируй канон**: если тема уже описана в `docs/*`, здесь оставляй только
  резюме, ссылки и “что изменилось”.
- **Не клади секреты**: токены, пароли, приватные URL и дампы — запрещены.
- **Именование**: `YYYY-MM-DD_<topic>_<owner>.md`, например:
  `2026-05-08_rls-bypass-grants_gulmira.md`.

## “Для кода тоже”

Если спринтовый документ вводит **временное поведение в коде** (фича‑флаг,
временный обход, деградация):

- **Обязательно** добавь в документ:
  - где в коде это находится (пути/символы),
  - условие включения (env/флаг/роль),
  - критерий удаления и дедлайн.
- В самом коде делай временные штуки максимально локальными и обратимыми
  (предпочитай явный фича‑флаг/конфиг, а не “скрытую” ветку логики).
